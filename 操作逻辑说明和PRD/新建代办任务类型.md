# 新建代办任务类型 操作逻辑说明与PRD

## 1. 文档目标
- 定义“新建待办”页面中“任务类型”模块与“选择任务类型”二级弹窗的完整交互规则。
- 约束当前三种类型（作业、考试、竞赛）的展示、选择、确认、回写逻辑。
- 预留未来扩展到 4+ 类型时的显示策略，避免首页标签区域臃肿。

## 2. 关联页面与入口
- 主页面：新建待办（全屏页面）。
- 当前模块：任务类型（主页面内胶囊标签行）。
- 二级界面：选择任务类型（居中弹窗）。
- 触发入口：主页面“任务类型”行最右侧灰色“...”胶囊容器。

## 3. UI结构定义

### 3.1 主页面任务类型行（一级）
- 可见位固定为 3 个：
  - 第1位：动态“已选任务类型”
  - 第2位：常见类型补位
  - 第3位：常见类型补位
- 最右侧“...”为弹窗入口，不参与选中状态。
- 选中态样式：黄色高亮容器（浅黄底 + 深黄文字）。
- 非选中态样式：灰色容器（浅灰底 + 深灰文字）。

### 3.2 选择任务类型弹窗（二级）
- 顶栏左上：关闭键（X），用于取消并返回。
- 顶栏标题：选择任务类型。
- 搜索框占位文案：搜索任务...
- 列表项（当前版本仅 3 项）：
  - 作业
  - 考试
  - 竞赛
- 弹窗底部：确认按钮。

## 4. 当前版本范围
- 有效任务类型仅包含：作业、考试、竞赛。
- 不展示“硕士”“通用”等历史占位项。
- 弹窗列表可滚动能力保留（为未来扩展预留），但在当前三项下无需滚动。

## 5. 交互流程

### 5.1 打开弹窗
1. 用户点击主页面任务类型行中的“...”
2. 页面上浮出“选择任务类型”弹窗
3. 主页面进入遮罩态，不可直接操作

### 5.2 选择类型（弹窗内）
- 用户点击任意类型行，仅更新“候选选中值（pendingType）”。
- 候选选中值在弹窗内可有视觉反馈（例如行高亮、勾选图标），但不立即提交到主页面。

### 5.3 确认提交
1. 用户点击“确认”
2. 将 pendingType 提交为“当前任务类型（selectedType）”
3. 关闭弹窗，返回主页面
4. 主页面任务类型行立即重排并刷新高亮

### 5.4 关闭取消
- 用户点击左上关闭键（X）或点遮罩关闭（若后续开启此能力）：
  - 不提交 pendingType
  - 保持 selectedType 不变
  - 关闭弹窗返回主页面
- 用户双击弹窗外空白区域：等价于取消并返回主页面。

## 6. 主页面重排规则（重点）

### 6.1 三槽位固定策略
- 主页面任务类型可视区永远只显示 3 个类型胶囊，避免版面拥挤。
- 当 selectedType 在常见集合内（作业/考试/竞赛）时：
  - 第1位显示 selectedType（黄色高亮）
  - 其余两项按预设顺序补齐

### 6.2 扩展类型置顶策略（必须）
- 当未来用户选中了“第4类及之后”的类型（例如：展示）：
  - “展示”必须进入主页面任务类型第1位
  - 且该第1位必须为黄色高亮
  - 原常见三类按顺序后移，仅保留两类在第2/第3位
- 目的：
  - 用户刚选中的类型可立即被看见
  - 不增加额外行数，不破坏当前紧凑布局

### 6.3 “...”入口常驻规则
- 无论 selectedType 是常见类型还是扩展类型，“...”入口始终保留在任务类型区域末端，用于再次打开弹窗。

## 7. 数据模型建议
- `selectedType: string` 当前已确认类型
- `pendingType: string | null` 弹窗临时选择
- `commonTypes: string[] = ["作业", "考试", "竞赛"]`
- `visibleTypes: string[]` 主页面三槽位最终渲染结果

推荐渲染计算：
1. 先放入 `selectedType`
2. 从 `commonTypes` 依次补齐，跳过重复项
3. 截断到3项

## 8. 状态机
- `idle`：主页面正常态
- `modal_open`：弹窗打开
- `type_pending`：弹窗内已选择但未确认
- `type_committed`：点击确认后提交成功并回到主页面

状态迁移：
- `idle -> modal_open`（点击“...”）
- `modal_open -> type_pending`（点列表项）
- `type_pending -> type_committed`（点确认）
- `modal_open/type_pending -> idle`（点关闭，取消）
- `type_committed -> idle`（渲染完成）

## 9. 边界与异常处理
- 重复点击同一类型后确认：允许；仅触发一次刷新，不产生重复项。
- 弹窗打开期间切后台：返回前台仍保留 pendingType（可选策略，建议保留）。
- 搜索无结果：显示空态文案“未找到相关任务类型”。
- 类型文案过长：单行省略号，不允许换行挤压列表高度。

## 10. 事件定义
- `onOpenTaskTypeModal()`
- `onSearchTaskType(keyword)`
- `onSelectTaskTypePending(type)`
- `onConfirmTaskType()`
- `onCancelTaskTypeModal()`
- `onRenderVisibleTaskTypes(selectedType)`

## 11. 验收标准
- 关闭键位于弹窗左上角。
- 搜索占位文案为“搜索任务...”。
- 列表中仅有“作业/考试/竞赛”三项。
- 点击确认后弹窗关闭并返回主页面。
- 主页面始终只显示3个任务类型胶囊。
- 未来选中第4类任务时，该类型必须置顶到第1位并高亮。
