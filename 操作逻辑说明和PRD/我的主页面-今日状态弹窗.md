# 我的主页面-今日状态弹窗 操作逻辑说明与PRD

## 1. 文档目标
- 定义“今日状态弹窗”与“今日状态弹窗激活态”的交互流程、状态迁移、数据回写规则。
- 统一“选择已有心情”与“新建心情”两条路径，避免用户心智冲突。
- 为后续 Coding 提供可直接实现的字段、事件和验收标准。

## 2. 页面范围
- 页面 A：我的主页面-今日状态弹窗（普通态）
- 页面 B：我的主页面-今日状态弹窗（激活态）
- 入口页面：我的主页面统计卡第 3 项「今日状态」

## 3. 入口与退出

### 3.1 打开弹窗
- 用户点击“我的主页面”中的「今日状态」分区。
- 弹出卡片式模态窗，背景加暗遮罩。

### 3.2 关闭弹窗
- 点击左上关闭键（X）可关闭。
- 点击弹窗外遮罩区域可关闭。
- 双击弹窗外空白区域可关闭并返回上一界面。
- 关闭后的回写规则见第 7 节。

## 4. 视觉与结构规范

### 4.1 标题区
- 左上：关闭键。
- 标题：今日状态。
- 标题需与关闭键处于同一水平线，且在弹窗内容区水平居中（当前已调整）。

### 4.2 搜索区
- 搜索占位文案：搜索心情...
- 功能：按关键字过滤心情选项。

### 4.3 心情列表区
- 可见区域固定展示 3 条心情卡片。
- 列表容器支持纵向滚动，查看更多心情。
- 视觉提示：
  - 右侧浅灰细滚动指示条。
  - 底部显示“下一条心情顶部预览”（如“专注...”）提示可滑动。

### 4.4 新建入口
- 底部按钮：新建心情（黄色主按钮）。
- 点击后进入“激活态”。

### 4.5 激活态附加输入行
- 在列表区与底部按钮之间出现输入行：
  - 左侧灰色输入容器（可编辑文本）
  - 右侧黄色勾/叉（确认/取消）
- 输入容器默认文案：自定义名称

## 5. 默认选中与状态样式
- 默认状态值：很棒。
- 选中项样式：浅黄底高亮容器。
- 未选中项样式：白色容器。

## 6. 交互流程

### 6.1 选择已有心情
1. 用户点击某条心情（例如：平静）。
2. 该项立即变为黄色高亮，其他项恢复白色。
3. 系统更新 `pendingMood`。

### 6.2 关闭并提交
- 若用户在弹窗中已完成选择：
  - 点击关闭键或遮罩时，提交 `pendingMood` 为 `selectedMood`。
  - 回写我的主页面统计卡第 3 项文案。
- 例如：选择“平静”后关闭，主页显示“今日状态：平静”。

### 6.3 新建心情（激活态）
1. 点击“新建心情”进入激活态。
2. 用户点击灰色输入容器，唤起输入法。
3. 输入自定义文案后：
  - 点勾：校验通过则保存并加入心情列表。
  - 点叉：取消本次新建，不写入列表。
4. 保存成功后，新建心情可被选中并成为当前状态。

## 7. 数据回写规则
- 回写目标：我的主页面统计卡第三项 `todayMoodLabel`。
- 回写触发：
  - 选择已有心情后关闭弹窗。
  - 新建心情并确认后，若被选中则关闭时回写。
- 建议持久化粒度：按“日期维度”存储当日状态。

## 8. 数据模型建议
- `selectedMood: string` 当前已生效心情
- `pendingMood: string | null` 弹窗内临时选中值
- `moodList: string[]` 可选心情列表（含系统预置+自定义）
- `isMoodModalOpen: boolean`
- `isMoodCreateActive: boolean`
- `pendingCustomMood: string`

## 9. 校验规则
- 自定义心情判空：去首尾空格后不能为空。
- 长度建议：1~8 字（可扩展到 12）。
- 去重规则：与现有心情同名时禁止保存并提示。
- 非法字符：按产品策略过滤（可选）。

## 10. 状态机
- `modal_closed`
- `modal_open_normal`
- `modal_open_selecting`
- `modal_open_creating`
- `custom_mood_committed`
- `custom_mood_canceled`

关键迁移：
- `modal_open_normal -> modal_open_creating`（点击新建心情）
- `modal_open_creating -> custom_mood_committed`（点勾并校验通过）
- `modal_open_creating -> custom_mood_canceled`（点叉）
- `modal_open_selecting -> modal_closed`（点关闭/遮罩并提交）

## 11. 事件定义
- `onOpenMoodModal()`
- `onCloseMoodModal()`
- `onSelectMood(mood)`
- `onSearchMood(keyword)`
- `onOpenCreateMood()`
- `onInputCustomMood(value)`
- `onConfirmCustomMood()`
- `onCancelCustomMood()`
- `onCommitMoodToProfile(mood)`

## 12. 边界与异常处理
- 搜索无结果：显示“未找到相关心情”。
- 新建重名：提示“该心情已存在”。
- 新建失败：提示“保存失败，请重试”。
- 弹窗外关闭但无选择变更：保持原状态不变。

## 13. 验收标准
- 标题“今日状态”在弹窗内居中，且与关闭键同水平。
- 默认高亮项为“很棒”。
- 仅显示 3 条心情卡片，并可滚动查看更多。
- 点击“新建心情”进入激活态，出现输入栏+勾叉。
- 点击勾可新增心情；点击叉不保存。
- 选择“平静”后关闭弹窗，主页“今日状态”同步为“平静”。
